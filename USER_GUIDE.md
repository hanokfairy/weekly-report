# 주간보고 관리자 페이지 사용법

이 문서는 관리자 페이지(웹 화면)를 사용하는 방법만 다룹니다. API 키 발급이나 서버 설치 같은 기술적인 내용은 몰라도 됩니다.

## 접속하기

1. 사무실 내부망에 연결된 상태에서 브라우저(크롬 등)를 엽니다.
2. 담당자에게 안내받은 주소로 접속합니다 (예: `http://192.168.0.15:5000`).
3. 아이디/비밀번호를 입력해 로그인합니다. 계정이 없다면 담당자에게 요청하세요.

## 대시보드 (첫 화면)

로그인하면 관리 중인 병원 목록이 카드 형태로 보입니다. 각 카드에는:

- 병원 이름, GA4/Slack 연결 정보
- 이번 주 스마트플레이스 방문/예약/리뷰 수치
- 아래 버튼들:
  - **설정 수정**: GA4/네이버/블로그/Slack 연결 정보 변경
  - **실적 수치 편집**: 매주 손으로 입력해야 하는 숫자 입력
  - **지난 보고서 보기**: 과거에 보낸 보고서 확인
  - **지금 실행**: 지금 바로 보고서를 만들어서 Slack으로 전송

## 매주 해야 하는 일: 실적 수치 입력

자동으로 수집되지 않는 값(카페 침투, 영수증 리뷰, 그리고 스마트플레이스 자동 수집이 안 되는 경우엔 방문/예약 수치까지)은 매주 사람이 직접 입력해야 합니다.

1. 해당 병원 카드에서 **"실적 수치 편집"** 클릭
2. 이번 주 실제 값으로 숫자 칸을 채웁니다
   - 블로그: 이번 달 목표 건수 중 완료한 건수
   - 카페 침투 / 영수증 리뷰: 목표·완료 건수, 필요하면 메모
   - 스마트플레이스: 기간, 이번 주/전주 방문수, 예약 신청, 리뷰 등록 수
3. **"저장"** 버튼 클릭 → 자동으로 반영됩니다

## 새 병원 추가하기

1. 대시보드 상단 **"+ 새 클라이언트 추가"** 클릭
2. 아래 정보를 담당자(대행사 실무자)에게 확인해서 입력합니다:
   - 클라이언트 ID: 영문/한글 아무거나, 한 번 정하면 나중에 바꿀 수 없음
   - 병원 표시 이름
   - GA4 속성 ID
   - 블로그 정보 (블로그가 여러 개면 "블로그 2", "블로그 3" 칸까지 채우면 됩니다. 블로그마다 체크할 키워드를 다르게 넣을 수 있습니다)
   - Slack 채널 ID (봇을 그 채널에 먼저 초대해야 합니다 — `/invite @봇이름`)
3. **"저장"** 클릭

## 설정이 잘못됐을 때

- **"설정 수정"**에서 언제든 값을 고칠 수 있습니다. 블로그 칸의 이름/RSS 주소를 비워두고 저장하면 그 블로그는 삭제됩니다.
- 저장 버튼을 누르면 자동으로 기록되어, 다음 주 보고서부터 바로 반영됩니다.

## 지난 보고서 확인하기

해당 병원 카드에서 **"지난 보고서 보기"**를 클릭하면 날짜별로 과거에 보낸 보고서 전체 내용을 볼 수 있습니다.

## 지금 바로 보고서를 보내고 싶을 때

정기 실행(매주 월요일 아침)을 기다리지 않고 지금 바로 확인하고 싶다면 **"지금 실행"** 버튼을 누르세요. 확인 창이 뜨면 "확인"을 누르고 잠시 기다리면, 해당 병원의 Slack 채널로 `.txt` 파일이 도착합니다.

## 문제가 생겼을 때

- 로그인이 안 될 때: 아이디/비밀번호 오타 확인, 그래도 안 되면 담당자에게 계정 재발급 요청
- 페이지 접속이 안 될 때: 사무실 네트워크(Wi-Fi) 연결 확인, 그래도 안 되면 관리자 컴퓨터가 켜져 있는지 담당자에게 확인 요청
- 저장이 안 되거나 오류 메시지가 뜰 때: 화면에 뜨는 오류 메시지를 그대로 캡처해서 담당자에게 전달
